# Stage2E70 Family Analysis

Scope:

- `patch_stage2e70*`
- `full_redesign_stage2e70*`

This note compares the original stage2e70 families with the newly added `vitsmall_p64` and `vitbase_p{256,384,512}` variants. The goal is not only to rank the runs, but also to explain what each family is doing, why some local behaviors differ strongly while headline performance often stays in the same band, and which conclusions are actually robust.

The conclusions here are based on:

- `runs/patch_stage2e70*/.../evaluation/evaluation_metrics.json`
- `runs/full_redesign_stage2e70*/.../evaluation/evaluation_metrics.json`
- `experiments/stage2e70_no_divergence_sweep.json` and `experiments/stage2e70_no_divergence_sweep.csv`
- `training_summary.json` and `training_history.json` for both families
- `evaluation/cluster_diagnostics/*_summary.json` for the original redesign family
- representative family configs in `project/configs/`
- the existing comparison baselines in `PATCH_RETROSPECTIVE.md` and `REDESIGN_RETROSPECTIVE.md`

The important point is that the new families did not change the overall stage2e70 training envelope. They changed backbone, projection dimension, mode, and K coverage, but kept the same broad two-stage recipe.

## 2. How The Two Families Work

### 2.1 Redesign Family

`full_redesign_stage2e70*` uses semantic centroid anchors.

- anchors are generated by k-means in frozen DINOv3 embedding space
- anchors use `representation: centroids`
- anchors are reprojected each forward pass
- sufficiently large K values are pruned after clustering
- stage-1 labels are fixed and capacitated
- stage-2 aligns to the sample's fixed anchor label

Operationally, redesign is trying to learn a global prototype structure. That makes its post-hoc cluster diagnostics meaningful: margin, ratio, effective anchors used, and entropy all say something about the geometry of the learned assignment field.

### 2.2 Patch Family

`patch_stage2e70*` uses local dense patch-to-anchor matching.

- `anchor.mode: patch`
- anchors use `representation: closest_samples`
- image scores come from reduced patch-to-anchor distances, not only from a global prototype distance
- `train_augment_mode: none` is used to preserve spatial patch meaning
- stage-2 still runs under the same broad stage2e70 setup, but patch mode remains a local reference-bank detector first and a global clustering method second

Operationally, patch mode is not trying to tell the same semantic-cluster story as redesign. It is trying to provide a richer local normal reference bank. Similar AUROC between patch and redesign therefore does not mean they learned the same internal structure.

## 3. What Changed Across The New Variants

The new variants changed only a few knobs.

| Variant Group | What Changed | What Stayed Fixed |
| --- | --- | --- |
| `vitsmall_p64` | projection dim `128 -> 64` | backbone frozen, stage2e70 recipe, fixed pseudo-labels, same fusion weights |
| `vitbase_p256/p384/p512` | backbone `vit_small -> vit_base`, projection dim `256/384/512` | backbone still frozen, stage2e70 recipe unchanged, fixed pseudo-labels, same fusion weights |
| redesign vs patch | anchor mode and anchor representation | same data, same two-stage structure, same 70-epoch stage-2 budget with early stopping |

This matters for interpretation. The new comparison is not “which idea is best in the abstract?” It is “what happens if the same stage2e70 recipe is transplanted onto different backbone/projection settings?”

## 8. Signal Decomposition: Why Fused AUROC Can Move Without Image AUROC Moving Much

The most informative runs are the ones where fused AUROC and anchor/image AUROC disagree.

| Run | Anchor Image | Reconstruction Image | Divergence Image | Pixel-Agg Image | Patch-Div Image | Fused Image |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `full_redesign_stage2e70_k1` | `0.8057` | `0.5601` | `0.4702` | `0.6418` | `0.4955` | `0.7718` |
| `full_redesign_stage2e70_k1024` | `0.7685` | `0.7567` | `0.6871` | `0.7432` | `0.6978` | `0.8260` |
| `patch_stage2e70_k32` | `0.8119` | `0.7238` | `0.3809` | `0.7226` | `0.2574` | `0.8282` |
| `patch_stage2e70_vitbase_p384_k1` | `0.7518` | `0.7668` | `0.5161` | `0.7501` | `0.7404` | `0.8165` |

These four runs explain most of the apparent paradoxes.

### 8.1 Why redesign `K=1` still has the biggest image AUROC

`full_redesign_stage2e70_k1` wins on the anchor/image score because it is the cleanest one-center detector.

- there is no second anchor to create assignment ambiguity
- there is no fragmented large-K bank to weaken nearest-vs-second-nearest separation
- the anchor score alone is already strong enough to beat every other redesign image score

Its weakness is that the auxiliary signals are not strong. Reconstruction, divergence, and pixel-aggregated image scores are much weaker than in the better fused redesign runs. That is why it wins image AUROC but not fused AUROC.

### 8.2 Why redesign `K=1024` wins fused AUROC but not image AUROC

`full_redesign_stage2e70_k1024` does not beat `K=1` on anchor/image AUROC. It wins fused AUROC because the auxiliary signals become useful enough to push the combined score up.

- anchor image AUROC is only `0.7685`
- but reconstruction image AUROC is `0.7567`
- divergence image AUROC is `0.6871`
- pixel-aggregated image AUROC is `0.7432`
- patch-divergence aggregated image AUROC is `0.6978`

This is the large-K redesign regime: the anchor field is geometrically weak, but it provides enough coverage that stage-2 and fusion can exploit residual information.

### 8.3 Why patch `K=32` wins overall

`patch_stage2e70_k32` is different. It wins because the anchor-side signal itself is the strongest patch image detector.

- anchor/image AUROC is already `0.8119`
- reconstruction and pixel-aggregated scores are only moderate
- divergence is weak
- fused AUROC rises only modestly, from `0.8119` to `0.8282`

So patch `K=32` is not winning because fusion rescues a mediocre anchor detector. It is winning because the dense local reference bank is already a strong detector.

### 8.4 Why some new patch variants fuse well without strong anchor AUROC

`patch_stage2e70_vitbase_p384_k1` is the clearest example.

- anchor/image AUROC is only `0.7518`
- reconstruction image AUROC is `0.7668`
- pixel-aggregated image AUROC is `0.7501`
- patch-divergence aggregated image AUROC is `0.7404`
- fused AUROC jumps to `0.8165`

This run does not prove that the vit-base variant is globally better. It shows that under fixed fusion weights, a run can look strong in fused AUROC because several medium-strength signals align, even when its anchor detector alone is not top-tier.

### 8.5 What happens if divergence is omitted from fusion

The post-hoc sweep in `experiments/stage2e70_no_divergence_sweep.json` and `.csv` removes divergence from fusion and recomputes the image-level score with `0.5 * anchor + 0.5 * pixel_agg` on the saved best-model outputs. This is not a retrain. It is a controlled readout of how much divergence was helping the already saved runs.

| Family | Runs | Best No-Div Run | Best No-Div AUROC | Mean Delta vs Stored Fused | Improved Runs | Selected Divergence Counts |
| --- | ---: | --- | ---: | ---: | ---: | --- |
| redesign | `13` | `full_redesign_stage2e70_vitsmall_p64_k1` | `0.8237` | `+0.0151` | `7/13` | `CLS 5`, `patch 7`, `drop 1` |
| patch | `18` | `patch_stage2e70_k32` | `0.8225` | `+0.0073` | `8/18` | `CLS 1`, `patch 9`, `drop 8` |

- In redesign, removing divergence helps more often than not, especially in several `K=1` and mid-K runs where the divergence branch is weak or only barely above the anti-correlation threshold. The clearest examples are `full_redesign_stage2e70_vitbase_p512_k1` (`0.7010 -> 0.7786`), `full_redesign_stage2e70_vitbase_p384_k1` (`0.7205 -> 0.7866`), and `full_redesign_stage2e70_vitbase_p256_k1` (`0.7332 -> 0.7892`).
- The strongest large-K redesign runs still show why divergence was kept in the original fusion recipe. `full_redesign_stage2e70_k1024` drops from `0.8260` to `0.8118`, `full_redesign_stage2e70_vitsmall_p64_k1024` drops from `0.8098` to `0.7783`, and `full_redesign_stage2e70_vitbase_p512_k1024` drops from `0.8188` to `0.7913` when divergence is removed.
- In patch mode, divergence is frequently either selected from the patch branch or dropped entirely. That matches the family’s behavior: patch is often a strong anchor-plus-pixel detector first, with divergence as an occasional helper rather than the main driver.
- The best original patch baseline, `patch_stage2e70_k32`, barely changes when divergence is removed (`0.8282 -> 0.8225`). That strengthens the interpretation that its win is mostly anchored in dense local reference matching plus the reconstruction-side pixel signal, not in a strong divergence term.

## 9. Why Internal Behavior Changes More Than Overall Performance

This is the main interpretation.

### 9.1 The pixel branch is consistently strong across almost everything

Pixel AUROC stays in a narrow range.

- redesign: roughly `0.9094` to `0.9332`
- patch: roughly `0.9117` to `0.9351`

That means the reconstruction-side anomaly map is a stable backbone of performance. Many config changes alter the anchor-side and geometry-side behavior more than they alter the pixel branch.

### 9.2 The changed knobs mostly affect the anchor-side signal

Backbone, projection dimension, and K mostly change:

- anchor discrimination
- assignment confidence or ambiguity
- how much stage-2 and fusion can add on top of the anchor score

They do not create a completely new detector. They perturb a detector that is already anchored by strong frozen DINO features and a robust reconstruction path.

That is why many runs move visibly in auxiliary metrics without creating a dramatic jump in final AUROC.

### 9.3 Frozen backbones limit how much vit-base can help

The new vit-base families are not a full vit-base redesign of the recipe. They are the same stage2e70 recipe transplanted onto a larger frozen backbone with larger projection dimensions.

That means:

- the recipe is still tuned around the original family assumptions
- `loss.margin: 1.0` and the fixed fusion weights were not retuned for larger latent spaces
- the larger feature space can increase ambiguity or calibration mismatch instead of automatically improving detection

The right conclusion is not “vit-base is worse.” The right conclusion is:

- vit-base under this unchanged frozen stage2e70 recipe did not beat the established vit-small baselines

### 9.4 Lower projection dimension can stay competitive because the system is not bottlenecked there

The `vitsmall_p64` runs are informative.

- redesign `vitsmall_p64_k1` still reaches `0.7922` image AUROC and `0.7867` fused AUROC
- patch `vitsmall_p64_k1` still reaches `0.7790` image AUROC

So halving the projection dimension does change the anchor geometry, but it does not collapse the system. The strongest evidence is that the pixel and fused signals remain broadly competitive. The detector is not operating in a regime where “more projection dimension” is automatically the dominant factor.

## 10. Family-Level Conclusions

### 10.1 Redesign Family

1. The original `vit_small`, `projection_dim=128`, `K=1` run still has the best redesign image AUROC.
2. The best redesign fused AUROC still comes from large K, specifically `K=1024`.
3. The new redesign variants changed the training dynamics more than the leaderboard.
4. `vitsmall_p64_k1` is the strongest new redesign challenger and is surprisingly close to the original `K=1` image result.
5. Some vit-base `K=1024` variants are competitive in fused AUROC, but none beat the original `full_redesign_stage2e70_k1024`.
6. Under the no-divergence `0.5/0.5` sweep, `7/13` redesign runs improve and the best redesign no-divergence readout becomes `full_redesign_stage2e70_vitsmall_p64_k1` at `0.8237`, but the strongest large-K redesign baselines still lose when divergence is removed.

### 10.2 Patch Family

1. The original `vit_small`, `projection_dim=128` patch family remains the best practical detector, especially at `K=16` to `K=32`.
2. `patch_stage2e70_k32` is still the strongest patch run by both image AUROC and fused AUROC.
3. The new patch variants are competitive, and some improve training stability, but none displaced the original `k32` baseline.
4. The strongest new patch image run is `vitbase_p256_k1`, while the strongest new patch fused run is `vitbase_p384_k1`.
5. Patch mode remains a local reference-bank detector first. Its strongest results do not require redesign-scale K values.
6. Under the no-divergence `0.5/0.5` sweep, `8/18` patch runs improve, but the best original patch baseline `patch_stage2e70_k32` changes only slightly (`0.8282 -> 0.8225`), which confirms that it was already mostly an anchor-plus-pixel detector.

## 11. Recommendations

### 11.1 If the goal is best redesign image AUROC

Keep `full_redesign_stage2e70_k1` as the main redesign image-level baseline.

### 11.2 If the goal is best redesign fused AUROC

Keep large-K redesign in play, especially `full_redesign_stage2e70_k1024`, but describe it honestly as a fragmented reference-bank regime rather than as clean clustering.

### 11.3 If the goal is best patch detector

Use `patch_stage2e70_k16` and especially `patch_stage2e70_k32` as the patch baselines to beat. The new backbone/projection variants did not surpass them.

### 11.4 If the goal is fair backbone/projection comparison

Do not treat the current vit-base results as a final verdict on vit-base. First retune at least:

- margin or regularization strength
- fusion weights
- possibly backbone freeze policy

The current comparison kept those fixed on purpose, which made the comparison controlled but not fully optimized for the new latent spaces.

### 11.5 Across both families

Stage-1 checkpoint selection remains the main shared methodological problem. Many of the multi-anchor runs still either prefer epoch `1` or show strong sensitivity to early epochs. That should be addressed before drawing stronger conclusions about the intrinsic value of larger K, larger backbones, or larger projection dimensions.

## 12. Bottom Line

The new variants changed the internal behavior much more than they changed the final scoreboard.

- redesign still splits into a simple high-image-AUROC `K=1` regime and a higher-fused-AUROC large-K regime
- patch still works best in the original `vit_small p128` moderate/high-K region
- removing divergence from fusion improves many runs, but it does not erase the large-K redesign case where divergence is still genuinely useful
- vit-base and lower projection dimensions can be competitive, but under this unchanged frozen stage2e70 recipe they did not create a new best family

So the main lesson is not that the new variants failed. It is that the original stage2e70 families were already sitting in a strong performance band, and the new backbone/projection changes mainly redistributed where the signal comes from:

- anchor score versus fusion
- simple one-center behavior versus fragmented reference-bank coverage
- optimization stability versus final AUROC

That is exactly why the internal stories differ more than the headline performance does.
